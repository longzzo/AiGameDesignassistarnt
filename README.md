# AI Game Design & Balance Assistant

장르에 국한되지 않고 어떤 게임이든 기획 명세와 밸런싱을 데이터로 검증/자동화해 주는 범용 웹 대시보드. **Monorepo (frontend + backend)** 구조의 MVP 스캐폴드.

## 기술 스택

| 영역 | 스택 |
|---|---|
| 프론트엔드 | React 19 (Vite 8), TailwindCSS v4, Recharts 3, Axios |
| 백엔드 | Python 3.12, FastAPI, Pydantic v2, Uvicorn |
| 통신 | Vite dev proxy (`/api` → `127.0.0.1:8000`) |

## 모듈 / 엔드포인트

게임 개발 퍼널(**발상 → 기획 → 검증 → 생성**)을 따라 11개 모듈을 제공합니다.

| 모듈 | 엔드포인트 | 프론트 레이아웃 |
|---|---|---|
| 장르·기능 추천 (발상) | `POST /api/v1/ideation/recommend` | 입력폼 + 추천 카드 그리드 |
| 기획서(GDD) 생성 (기획) | `POST /api/v1/gdd/draft` | 입력폼 + 문서/메타프롬프트 탭 |
| 레벨 디자인 (기획) | `POST /api/v1/level/design` | 페이싱 곡선(+권장 곡선) + 난이도 점프 진단 + 비트시트 + 인카운터(AI 상세) + 비평 |
| 월드 바이블 검증 (기획) | `POST /api/v1/lore/validate` | 세계관/스토리/퀘스트 탭 + 일관성 이슈 |
| 개발 로드맵 (PM) | `POST /api/v1/plan/wbs` | WBS / 주차별 일정 / 기능명세서 탭 |
| 스탯 / DPS·TTK 시뮬레이터 | `POST /api/v1/simulator/dps-ttk` | Top-Bottom (가로폼 + 풀폭 꺾은선) |
| 경제 인플레이션 추적기 | `POST /api/v1/economy/inflation` | Grid Cards (3카드 + 막대 + 경고배너) |
| 가챠 몬테카를로 검증 | `POST /api/v1/gacha/monte-carlo` | 3:7 Split (폼 + 통계·히스토그램) |
| 밸런스 랩 (밸런싱) | `POST /api/v1/balance/solve` · `/balance/compare` | 2탭 (목표 역산 Goal-Seek / 빌드 비교 테이블) |
| 에셋 Lore 파이프라인 | `POST /api/v1/asset/lore-translate` | 5:5 Split (원본/다국어 JSON 에디터) |
| 프로토타입 프롬프트 생성 | `POST /api/v1/prototype/prompt` | 입력폼 + 다크모드 코드 블록 |

## 실행 방법

### 1) 백엔드 (포트 8000)

```bash
cd backend
# 최초 1회: 가상환경 + 의존성
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
# 서버 실행
.venv\Scripts\python.exe -m uvicorn main:app --reload --port 8000
```

- API 문서(Swagger): http://127.0.0.1:8000/docs

### 2) 프론트엔드 (포트 5173)

```bash
cd frontend
npm install      # 최초 1회
npm run dev
```

- 대시보드: http://localhost:5173
- `/api` 요청은 Vite 프록시가 자동으로 백엔드(8000)로 전달합니다.

> 두 서버를 동시에 띄운 뒤 대시보드에 접속하세요.

## AI 연결 (GitHub Models)

AI 기능은 **GitHub Models**에 연결됩니다(Anthropic/Azure 아님, OpenAI 호환). 토큰이 설정되면 실제 AI 응답을 사용합니다.

> **AI 호출 불가 시 동작**: 토큰이 없거나 호출이 실패/제한되면, **AI가 작성하는 결과**(코치 제안·GDD 본문·장르 추천·에셋 번역·프로토타입 프롬프트/HTML·기능 목록·레벨 비평/인카운터 상세·WBS 조언·로어 의미검증·밸런스 해석)는 가짜 목업 대신 **"현재 API를 호출할 수 없습니다"** 로 표시됩니다. 반면 **결정론적 계산**(DPS/TTK·경제·가챠·빌드 비교 시뮬레이션, 레벨 구조·페이싱 곡선·난이도 진단, 로어 규칙 검사, WBS 일정·기능명세서, 밸런스 역산 탐색)은 AI 없이도 그대로 동작합니다.

**모델 라우팅**
- **메인(`openai/gpt-5`)**: 프롬프트 생성, 시뮬레이션 피드백(AI 코치), 기획서(GDD), 장르 추천
- **미니(`openai/gpt-5-mini`)**: 단순 반복·번역(에셋 Lore)

**연결 방법**
1. `backend/.env.example` → `backend/.env` 복사 후 값 채우기:
   ```
   GITHUB_MODELS_TOKEN=<models 스코프를 가진 GitHub PAT>
   GITHUB_MODELS_ENDPOINT=https://models.github.ai/inference   # 선택
   LLM_MODEL_MAIN=openai/gpt-5                                  # 선택
   LLM_MODEL_MINI=openai/gpt-5-mini                             # 선택
   ```
2. 백엔드 재시작 → 헤더 배지가 **🟢 AI 연결됨 (GitHub Models)** 로 전환(미설정 시 🟡, AI 결과는 'API 호출 불가'로 표시).
3. 상태는 `GET /api/v1/meta`로도 확인.

> 구현: `backend/api/v1/services/llm.py` (공식 `openai` SDK의 `OpenAI` 클라이언트 + `base_url`, `tier="main"|"mini"`). gpt-5 계열 특성상 `temperature` 미전송·`max_completion_tokens` 사용. 호출 실패 시 `run_or_fallback`이 로그를 남기고 목업으로 폴백합니다.

## AI 활용 기능

- **🤖 AI 밸런스 코치** (`POST /api/v1/coach/suggest`): 스탯·경제·가챠 시뮬레이션 결과를 gpt-5에 넘겨 *"성장률 0.08→0.06"* 같은 구체적 수치 튜닝안을 받음. 각 시뮬레이터 하단의 "튜닝안 받기" 버튼.
- **🎮 실행 가능한 HTML 프로토타입** (`POST /api/v1/prototype/html`): 장르/메커니즘을 단일 HTML 게임으로 생성해 앱 내 iframe에서 즉시 플레이. 프로토타입 모듈의 "실행 HTML 생성" 탭.
- **검증 노트 보존**: 수집한 검증 노트(findings)는 localStorage에 저장되어 새로고침에도 유지됩니다.

## 분석 / 지속성 기능

- **가챠 50/50 천장(원신식)**: 확률 모듈의 체크박스로, 5성 획득 시 50% 비픽업·실패 시 다음 5성 확정 모델을 켜면 '픽업 획득까지'의 이중 봉우리 분포를 시뮬레이션합니다.
- **스탯 A/B 곡선 비교**: 스탯 모듈의 'B안 비교'로 대안 파라미터의 TTK 곡선을 같은 차트에 겹쳐 봅니다.
- **프로젝트 내보내기/불러오기**: 헤더의 ⬇/⬆ 버튼으로 작업 상태(localStorage `gg_*` 키)를 JSON 파일로 저장/복원합니다. 에셋 에디터 내용도 보존됩니다.
- **레벨 디자인 보강**: 레벨 유형/길이/메커니즘/난이도 곡선을 넣으면 **비트 시트 + 페이싱(긴장도) 곡선 + 인카운터 배치 + 디자인 비평**(난이도 스파이크·학습 모멘트·페이싱)을 생성합니다. 구조는 결정론, 비평은 gpt-5 강화.
- **AI 인카운터 상세화**: 토큰 연결 시 gpt-5가 각 인카운터 슬롯을 그 레벨의 메커니즘·강도에 맞춰 **적 구성(👾 예: '창병 4 + 궁수 2')과 배치·전술(📍 예: '좌측 고지대 점거, 협곡으로 유도')** 로 구체화합니다(토큰 없으면 '구간 중심' 요약으로 폴백). 위치·종류 골격은 결정론으로 고정하고 상세만 AI가 채워, 슬롯 개수·순서가 어긋나도 안전하게 병합합니다. 비평과 함께 1회 호출로 생성합니다.
- **레벨 정량 난이도 진단**: 인접 비트 간 긴장도 변화(Δ)를 계산해 **급상승(+25↑) 구간을 자동 경고**하고, 급경사를 완화한 **권장 곡선을 실제 곡선 위에 점선으로 겹쳐** 보여줍니다. 가장 큰 점프는 배지로, 구간별 변화는 색상 칩으로 표시합니다(결정론).
- **밸런스 랩 — 목표 역산(Goal-Seek)**: `"후반 TTK 3초"`, `"천장 사용률 25%"` 같은 **목표 수치를 넣으면 파라미터를 역산**합니다(이분법 탐색, 스탯/경제/가챠 지원). 수렴 과정 그래프 + 도달/미도달 판정 + 조언을 제공합니다. 탐색은 결정론, 해석은 gpt-5 강화. `POST /api/v1/balance/solve`.
- **밸런스 랩 — 빌드 비교**: 여러 무기/직업 빌드의 **DPS·TTK를 한 표에서 비교**하고 중위 대비 ×1.25↑를 **OP**, ×0.8↓를 **약함**으로 자동 판정합니다(색상 막대 차트 + 격차 요약). `POST /api/v1/balance/compare`.
- **월드 바이블 일관성 검증**: 캐릭터·진영·장소·사건·퀘스트 엔트리를 누적하고, **규칙 기반(토큰 없이 동작)** 으로 중복 이름·미정의 참조·퀘스트 순환/누락·타임라인 충돌을 검출합니다. 토큰 연결 시 gpt-5가 의미적 모순(플롯 홀·톤)까지 검사.
- **세션(프로젝트) 저장**: 우측 **프로젝트 패널**(＋새 프로젝트 · 최근 항목 · 클릭 전환 · 💾저장 · 삭제, 접기 가능)으로 여러 프로젝트를 나눠 저장·전환합니다. 월드 바이블·검증 노트·에셋·로드맵 입력이 프로젝트별로 기억됩니다.
- **개발 로드맵(WBS)**: 기간(주)·**역할 구성(기획/개발/아트/QA 인원)**·기능 목록을 넣으면 **역할별 WBS 작업 분해 + 역할 용량 기반 주차별 일정 + 기능명세서**를 생성합니다. 일정은 **주차별 역할 누적 막대 차트**(가용 기준선)로 시각화하고, 역할별 요약 카드와 초과 경고를 제공합니다. **작업 분해는 토큰 연결 시 gpt-5가 기능 성격에 맞춰** 만듭니다 — 예: '사운드'는 음원 제작·오디오 통합·믹싱 QA, '멀티플레이'는 서버 구현·상태 동기화·부하 테스트처럼 기능마다 다른 작업/역할이 나옵니다(토큰 없으면 표준 설계→아트→구현→테스트 템플릿으로 폴백). AI가 만든 작업도 역할은 입력 역할 목록으로 제한하고 의존성은 유효한 DAG로 고정해 스케줄러와 안전하게 맞물립니다. PM 조언도 gpt-5 강화.
- **기능 목록 자동 생성**: 한 줄 설명을 넣고 "✨ 기능 목록 생성"을 누르면 개발 단위 기능 목록이 채워집니다(`POST /api/v1/plan/features`, 규칙 기반 + gpt-5 강화).
- **Trello용 CSV 내보내기**: 로드맵 탭의 "📄 CSV 내보내기 (Trello)"로 WBS를 카드 단위 CSV(`List`=주차, `Title`=작업, `Labels`=페이즈, 공수·선행 포함, UTF-8 BOM)로 받습니다. 이 CSV를 Claude + Trello MCP에 넘겨 보드 카드로 생성할 수 있습니다.

## 비고

- `simulator`, `economy`, `gacha`는 실제 수학/통계 로직(성장 곡선·Faucet/Sink 균형·몬테카를로)을 구현했습니다.
- `ideation`은 큐레이션된 장르 카탈로그 + 키워드 매칭 점수화로 동작합니다(결정론적). `gdd`는 표준 GDD 아웃라인을 결정론적으로 채운 초안 + LLM 확장용 메타 프롬프트를 함께 냅니다. `asset/lore-translate`, `prototype/prompt`도 템플릿/Mockup 로직이며, 운영 단계에서 LLM 호출로 교체하는 지점입니다.
- 에셋 파이프라인의 **세계관(`world_setting`)은 선택 입력**입니다. 비우면 순수 번역, 채우면 톤 & 매너가 입혀진 현지화 모드로 동작합니다(응답의 `lore_applied` 플래그로 구분).
- **발상 → 기획 연결**: 아이데이션 추천 카드의 "이 장르로 기획서 만들기 →" 버튼을 누르면 해당 장르·핵심 기능·레퍼런스가 GDD 모듈로 전달되어 기획서 초안이 자동 생성됩니다.
- **기획 → 검증 연결**: GDD 응답에는 장르/플랫폼에 맞춘 `balance_baseline`(스탯·경제·가챠 제안 시작값)이 포함됩니다. GDD 화면의 "⚔️ 스탯 / 💰 경제 / 🎲 가챠" 검증 버튼을 누르면 해당 값이 시뮬레이터 폼에 자동 주입되고 즉시 실행됩니다.
- **검증 → 기획 역방향 루프**: 각 시뮬레이터의 AI 피드백 옆 "📌 기획서에 메모" 버튼으로 발견사항을 모은 뒤, GDD 화면의 "노트 반영해 재생성"을 누르면 `validation_notes`가 기획서의 **★ 밸런스 검증 노트** 섹션으로 반영됩니다. 이로써 **발상 → 기획 → 검증 → 재기획**이 한 바퀴 닫힙니다.
- **초보자 가이드**: 사이드바 좌측 하단의 "💡 사용 가이드" 패널이 현재 모듈에 맞춰 *무엇인지 / 어떻게 쓰는지(3단계) / 초보자 TIP*을 보여줍니다(레벨 디자인·밸런싱 입문자용). 콘텐츠는 [`frontend/src/data/guides.js`](frontend/src/data/guides.js)에서 관리합니다.
